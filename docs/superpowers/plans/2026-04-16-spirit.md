# Spirit 精灵 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build an independent TypeScript MCP Server + Log Watcher service that monitors logs, analyzes errors with Claude, and auto-creates GitLab MRs via git worktree isolation.

**Architecture:** Two-process design — MCP Server (stdio, 18 tools) for IDE integration + Log Watcher (long-running daemon) for automated error detection and fix. Shared modules for ES, Git, GitLab API, SQLite incident store.

**Tech Stack:** TypeScript, @modelcontextprotocol/sdk, @elastic/elasticsearch, simple-git, chokidar, @anthropic-ai/sdk, better-sqlite3, yaml, zod

### Task 17: Watcher — Auto-Fix Pipeline

**Files:**
- Create: `src/watcher/auto-fix.ts`

- [ ] **Step 1: Implement auto-fix pipeline**

Function `autoFix(params: { incident, analysis, config })`:
1. Check risk level — if C, skip (just update incident with analysis)
2. Determine target repo from `incident.service` + config `codeRoots`
3. Create branch: `spirit/fix-{incident.id.slice(0,8)}-{timestamp}`
4. Create worktree
5. Apply code changes from `analysis.fixPlan`
6. Commit with message: `fix(spirit): {incident.title}\n\nIncident: {incident.id}\nRisk: {analysis.riskLevel}`
7. Push branch
8. Create GitLab MR with description containing:
   - Error summary
   - Sample logs
   - Claude analysis
   - Risk level
   - Auto-generated by Spirit 精灵
9. Update incident: status=fixing, branch, mr_url, mr_iid
10. Clean up worktree

Concurrency: use a semaphore (simple counter) to limit to `maxConcurrentFixes`.
